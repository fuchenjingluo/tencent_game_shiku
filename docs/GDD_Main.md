## 1. 设计支柱

> 每一个设计决策，都必须能回溯到至少一根支柱。  
> 如果某个功能对四根支柱都没有贡献——砍掉它。

### 🏛️ 支柱一：**「每一笔都有重量」— 有意义的道德困境**

| | |
|---|---|
| **玩家体验** | 没有"正确"答案。专业路线不一定最好，激进路线不一定最坏。每次选择都是一次价值取舍。 |
| **设计原则** | 三选一风格（专业/妥协/激进）中，没有任何一个在所有场景下都是最优解。gameFlags 因果链确保选择产生涟漪效应。 |
| **反模式** | 某个选择在超过 70% 的路径中都是明显最优（violates meaningful choice） |
| **当前状态** | ✅ 三选一架构完整。⚠️ 部分激进选项的风险惩罚可能过度，导致该路径在实际 playtest 中成为"从不选择"的选项 |

**关键度量**: 在 10 次 playtest 中，每种选择风格的使用率应在 20%-50% 之间。

### 🏛️ 支柱二：**「千年在你手中」— 沉浸式的文物保护叙事**

| | |
|---|---|
| **玩家体验** | 玩家不是在"打游戏"，而是在"做文保"。每一个小游戏背后都有真实的文物保护操作原型（红外扫描、湿度校准、注浆加固）。 |
| **设计原则** | `narrativeBinding` 字段将所有机制操作绑定到文物保护叙事。风险事件基于真实的石窟病害类型（起甲脱落、盐析、CO₂腐蚀）。 |
| **反模式** | 出现任何架空/奇幻元素，破坏真实感 |
| **当前状态** | ✅ 叙事绑定完整。⚠️ 缺少环境叙事（environmental storytelling），洞穴中应该有更多可交互的"坏境细节"而非纯装饰 |

**关键度量**: 玩家在体验后能说出的真实文保知识 ≥ 2 条。

### 🏛️ 支柱三：**「看不见的崩塌」— 风险是活物**

| | |
|---|---|
| **玩家体验** | 风险不是进度条上冰冷的数字。风险会爆炸——在你最意想不到的时刻。它像一个潜伏的敌人，每次操作都可能触发连锁反应。 |
| **设计原则** | 风险爆炸系统（Risk Explosion）：风险等级决定随机事件触发概率。safe=0% / warning=10% / danger=30% / crisis=100% per step。风险事件有轻重三级池。 |
| **反模式** | 玩家可以通过简单规避让风险永远不触发 / 或者风险永远无法被管理 |
| **当前状态** | ✅ 风险爆炸核心稳定。⚠️ 缺少"可预期的紧张感"——玩家看不到风险阈值线，无法做出"我知道风险要爆了所以赶紧处理"的策略决策 |

**关键度量**: 单次流程中至少触发 2-3 个风险事件；每次触发时玩家不感到"随机惩罚"，而是"我早该处理的"。

### 🏛️ 支柱四：**「策略深度在属性间流动」— 四维经济**

| | |
|---|---|
| **玩家体验** | 声誉、风险、证据、预算——四个属性之间可以转化。证据可以掩盖风险，预算可以购买声誉。但每一次转化都是不可逆的一次性决策。 |
| **设计原则** | 6 个一次性转化操作。每个都在不同维度间创造了策略空间。预算从初始 10 降低到 6，增加了资源稀缺感。 |
| **反模式** | 某个属性变得无关紧要（dead stat）/ 或某个转化在所有路径中都是必选的 |
| **当前状态** | ✅ 转化系统存在。⚠️ 缺少 HUD 层面的"转化时机提示"系统，玩家可能在整个流程中忘记使用转化。⚠️ 证据是唯一的多余属性——上限 100 但即使玩得好也很少突破 60，缺乏 sink 设计 |

**关键度量**: 每个 playtest 中至少有 3 个转化被使用；四种属性的最终值方差 ≥ 15%（避免所有属性趋同）。

## 2. 核心游戏循环

### 2.1 Moment-to-Moment（0–30 秒）

```
玩家操作                   反馈                     奖励
─────────────────────────────────────────────────────────
WASD/方向键移动           角色弹跳动画 + 脚印粒子     探索感
接近 NPC/交互点            [E] 提示弹出               目标发现感
按下 E 交互                对话框/选择面板弹出         叙事推进
小游戏操作                 视觉+音效实时反馈           操作满足感
```

**"Fun Hypothesis"（乐趣假说）**:  
> *玩家在昏暗的千年石窟中行走，每一次接近交互点都是一次悬念——"这次会遇到什么？"——而每一次小游戏操作都是"我在亲手保护文物"的具身体验。*

**当前状态评估**: ✅ 移动→交互→小游戏的流程流畅。⚠️ **缺失**：「什么都不做时」的反馈——玩家在房间里闲逛时缺少环境叙事触发（偶发的墙壁滴水声、壁画碎屑飘落粒子、蝙蝠飞出等氛围事件）。

### 2.2 Session Loop（15–25 分钟）

```
接收任务 → 前往监测点(Step1) → 三选一决策 → 小游戏 → 
前往设备/资料点(Step2) → 三选一决策 → 小游戏 → 
风险爆炸检查 → 任务完成
                                              ↓
                                    （循环 × 5个任务）
```

**张力来源**:  
1. **预算递减**: 初始 6 点预算，每个任务平均消耗 2-4 点，玩家在任务 3-4 时面临严重的资源焦虑
2. **风险递增**: 激进选择降低短期风险但增加长期隐患（gameFlags 后果）
3. **时间/操作压力**: 小游戏有时间限制，失败没有重来

**当前状态评估**: ✅ 循环结构完整。⚠️ **缺失**：Session Loop 的"高潮点"——任务 5 作为终章应该在数值和叙事上都给玩家一个"我已经到终点了"的信号。目前缺少：
- 任务 4→5 之间的"过渡叙事"（比如院长的紧急召唤、洞穴异常信号）
- 任务 5 完成后缺少"anecdote generator"——玩家没法分享"我是怎么拿到 S 的"的独特故事

### 2.3 Long-Term Loop（多周目）

```
第一周目结束 → 获得评级(S/A/B/C/D) + 评论
                    ↓
              解锁"再次挑战"按钮
                    ↓
        第二周目：尝试不同选择路径
        探索 gameFlags 因果链的另类结局
                    ↓
        追求全成就 / 全结局 / 速通排行榜
```

**当前状态评估**: ⚠️ **严重缺失**。目前只有单周目 + 重新开始的循环。缺少：
- **多结局系统**: 只有评级不同，没有真正不同的结局叙事
- **元进度（Meta-progression）**: 二周目没有任何可继承/可解锁的内容
- **锚定机制**: 没有 daily challenge、speedrun 计时器、或"本周最佳巡检员"排名

详见 [GDD_Replayability.md](./GDD_Replayability.md)

## 3. 难度曲线设计

### 3.1 当前曲线分析

```
任务      difficulty 范围    小游戏难度分布           预算压力   风险压力
─────────────────────────────────────────────────────────────────
Task 1    教学关              2×diff1, 2×diff1         低         中
Task 2    1~2                 1×diff1, 1×diff2,       中         中高
                              1×diff2, 1×diff2
Task 3    2                   2×diff2, 1×diff1,       中高       高
                              1×diff2, 1×diff3
Task 4    2~3                 1×diff2, 1×diff1,       高         高
                              1×diff3, 1×diff2
Task 5    3                   2×diff3, 1×diff1,       高         极高
                              2×diff2, 1×diff3
```

### 3.2 问题诊断

| 问题 | 严重度 | 说明 |
|---|---|---|
| Task 3 Step 2 的 diff 跳跃不均 | ⚠️ 中 | Step1 有 diff1 memory（简单），Step2 突然跳到 diff3 match，玩家可能感到"难度墙" |
| Task 5 Step 1 的 diff1 sequence | ⚠️ 中 | 终章却出现 diff1（简报排序），节奏感断裂 |
| 小游戏成功阈值未区分玩家技能 | 🔴 高 | Threshold 仅按 difficulty 分档（0.5/0.6/0.7），但没有考虑"玩家的历史表现"来动态调整 |
| 缺少"受挫→恢复"节奏 | ⚠️ 中 | 连续失败没有安全网设计，可能导致玩家在 Task 3-4 时放弃 |

### 3.3 优化建议

**难度曲线重设计**:
```
Task 1 (教学关):   diff1 × 4 — 100% 安全区
Task 2 (攀升):     diff1 × 1, diff2 × 3 — 引入压力和复杂操作
Task 3 (高原):     diff2 × 4 — 纯 diff2，一致性体验
Task 4 (再攀升):   diff2 × 2, diff3 × 2 — 引入最高难度
Task 5 (终章):     diff3 × 3, diff2 × 1 — 挑战巅峰，但最后一步留一个"滑翔"式的 diff2
```

**"安全网"设计**:
- 同一任务内连续两次小游戏失败 → 第三次的小游戏 diff 临时 -1
- 这个规则在 Task 1 不生效（教学关不需要）

## 4. 登船与教学流程

### 4.1 当前状态审计

| 检查项 | 状态 | 说明 |
|---|---|---|
| 核心操作在 30 秒内引入 | ✅ | WASD 移动在 BootScene 加载后立即可用 |
| 首次操作保证成功 | ❌ | Task 1 Step 1 Choice A (trace diff1) 是最简单的路径，但玩家可能选 B/C |
| 新机制在安全环境中引入 | ⚠️ | 风险爆炸、属性转化、gameFlags 都未在 Task 1 中明确教学 |
| 至少一个机制通过探索发现 | ❌ | 所有机制都通过对话框/NPC 教学，玩家没有"自己发现"的时刻 |
| 首次游戏以钩子结束 | ❌ | 游戏结束→评级报告→重新开始，没有"下一局会不一样"的悬念 |

### 4.2 登船优化方案

```
阶段 1: 移动教学（0-20秒）
  - BootScene 后，玩家在窟前栈道
  - HUD 提示："[WASD] 移动  [E] 交互"
  - 前方 3 格处有一个发光的"入门手册"交互点
  - 玩家走过去 → 按E → 触发简短角色自述（你是谁、今天要干什么）
  - ✅ 核心操作在 20 秒内引入

阶段 2: 首次任务 = 安全教学关（Task 1）
  - Task 1 明确标注为"张主任安排的入职训练"
  - 叙事提示"放心，这次任务不会有严重后果"
  - 6 个选项中至少 3 个标记 [推荐] 引导
  - 风险爆炸在 Task 1 中禁用（safe 状态不触发）
  - ✅ 首次成功几乎保证

阶段 3: 属性转化教学（Task 2 开始前）
  - 完成 Task 1 后，系统弹出属性面板
  - 高亮显示 6 个转化按钮
  - 文案："你的属性可以相互转化——但每个转化只能用一次。慎重选择。"
  - ✅ 新机制在安全时机引入

阶段 4: 发现式机制（Task 3 期间）
  - 在 Task 3 Step 1 的监测点附近，放一个"隐藏文件"交互点
  - 读取后揭示：风险超过 60 时事件触发概率暴增
  - 这不是 NPC 教的——是玩家自己找到的
  - ✅ 至少一个机制通过探索发现

阶段 5: 终章钩子（游戏结束）
  - 评级报告最后加一段：
    "不同的选择会揭示不同的结局。
     你选择了 [主要风格] 路线。
     完成度：[X]% —— 还有 [Y] 个因果分支等待探索。"
  - ✅ 游戏以"再来一局"的欲望结束
```

## 6. 玩家动机架构

### 6.1 动机映射 (Self-Determination Theory)

| 动机类型 | 当前满足度 | 对应的设计元素 |
|---|---|---|
| **自主性 (Autonomy)** | ⭐⭐⭐⭐ | 三选一 + 6 个转化方向 + 多种路径 |
| **胜任感 (Competence)** | ⭐⭐⭐ | 小游戏技能检验 + 评级系统 |
| **关联性 (Relatedness)** | ⭐⭐⭐ | NPC 角色塑造 + 石窟保护的主题共鸣 |

### 6.2 行为经济学注入点

| 机制 | 心理学原理 | 当前应用 | 增强建议 |
|---|---|---|---|
| 风险爆炸 | 损失厌恶 (Loss Aversion) | ✅ 已应用 | 让玩家在风险爆炸前有 1 步的"预感"窗口 |
| 属性转化（一次性） | 机会成本 + 禀赋效应 | ✅ 已应用 | 增加"转化预览"——显示转化后的预估属性值 |
| gameFlags | 承诺一致性 | ✅ 已应用 | 让玩家可以命名自己创建的 flag 路径（如"保守派路线"） |
| 评级系统 | 目标梯度效应 | ⚠️ 薄弱 | 将 S 评级拆分为 S- / S / S+ 增加目标逼近感 |
| 小游戏 | 变比率强化 | ✅ diff 随机化 | 增加稀有奖励小游戏（概率 5%，完成获得额外转化机会） |
