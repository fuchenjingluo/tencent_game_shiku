## 1. 设计理念

### 核心原则

> **小游戏不是打断，是具身。**  
> 玩家不是"被弹出一个迷你游戏来消磨时间"——他们是在"亲手操作文物保护设备"。  
> 每一个小游戏的操作隐喻都对应一种真实的文保操作。

### 四条铁律

| # | 规则 | 违反时玩家的感受 |
|---|---|---|
| 1 | 操作隐喻必须可追溯 | "我不知道这跟文物保护有什么关系" |
| 2 | 难度递增必须平滑 | "刚才还行，这个怎么突然这么难？" |
| 3 | 失败不是终结 | "完了，我搞砸了，前面的选择也白费了" |
| 4 | 成功有独特的满足感 | "就这样？点一下就好了？" |

### 小游戏选择逻辑

当前：每个 choice 硬编码了一个 miniGame 配置。

**增强建议**: 引入"小游戏池"——专业选择倾向于 calibrate/trace/wire（精确操作型），激进选择倾向于 timing（反应型），妥协选择倾向于 memory/sequence（认知型）。让选择风格和小游戏类型产生共振。

## 2. 机制规范：7 种小游戏

### 2.1 Trace（路径描摹）

```
───────────────────────────────────────────────────────────
操作隐喻:  红外热像仪 / 等高线绘制 / 标记边界
核心操作:  用鼠标/手指沿预设路径描摹
成功判定:  描摹轨迹与目标路径的重合度 ≥ 阈值
叙事绑定:  校准仪器、标记病害边界、绘制数据图
───────────────────────────────────────────────────────────
```

**难度爬升规则**:

| 参数 | Difficulty 1 | Difficulty 2 | Difficulty 3 |
|---|---|---|---|
| 时间限制 | 8 秒 | 6 秒 | 5 秒 |
| 路径复杂度 | 6 段，1 个正弦波 | 8 段，2 个正弦波 | 10 段，3 个正弦波 |
| 路径振幅 | ±40px (低) | ±55px (中) | ±70px (高) |
| 成功阈值 | 50% 重合度 | 60% 重合度 | 70% 重合度 |
| 描摹线宽容差 | 18px | 14px | 12px |

**调优杠杆**:
- `timeLimit`: 时间压力
- `pathSegments`: 路径复杂度（段数越多，玩家需要更多方向变化）
- `pathAmplitude`: 路径弯度（振幅越大，越难保持手稳）
- `successThreshold`: 精度要求
- `toleranceRadius`: 容错范围（越窄越难）

**边界情况**:
- 玩家在画布外完成描摹 → 轨迹记录不完整，自动判定失败
- 玩家画了直线忽略所有弯道 → `calcScore()` 返回极低值但不会崩溃
- 触屏设备不支持压感 → 忽略压感，仅记录(x,y)

**当前状态**: ✅ 实现完整。`generatePath` 的复杂性通过 diff 参数控制。⚠️ diff2→diff3 的振幅跳跃可能偏大。

### 2.2 Memory（序列记忆）

```
───────────────────────────────────────────────────────────
操作隐喻:  快速勘察 / 记住文物状态 / 现场记录
核心操作:  短暂展示一组信息布局 → 隐藏 → 玩家凭记忆复原
成功判定:  记忆复原的正确率 ≥ 阈值
叙事绑定:  记住壁画受潮点/CO₂读数/游客提示要点
───────────────────────────────────────────────────────────
```

**难度爬升规则**:

| 参数 | Difficulty 1 | Difficulty 2 | Difficulty 3 |
|---|---|---|---|
| 记忆项数量 | 4 个 | 6 个 | 8 个 |
| 展示时间 | 5 秒 | 3 秒 | 2 秒 |
| 记忆类型 | 颜色序列 | 位置+颜色 | 位置+颜色+形状 |
| 干扰项 | 无 | 2 个干扰 | 4 个干扰 |
| 成功阈值 | 75% 正确 | 75% 正确 | 70% 正确 |

**调优杠杆**:
- `itemCount`: 需要记忆的项数量
- `displayTime`: 展示窗口时长
- `complexity`: 每项的属性维度（1D=颜色, 2D=位置+颜色, 3D=位置+颜色+形状）
- `distractorCount`: 干扰项数量（增加选择难度）
- `threshold`: 正确率要求

**边界情况**:
- 玩家在展示期间切出应用 → 计时器继续运行，返回时可能已超时
- 色盲玩家 → 使用形状+颜色双重编码（当前未实现，建议增加）

**当前状态**: ✅ 基础实现。⚠️ 缺少干扰项设计（distractorCount），当前仅靠增加 itemCount 提升难度。

---

### 2.3 Match（翻牌配对）

```
───────────────────────────────────────────────────────────
操作隐喻:  设备选型 / 工具匹配 / 数据关联
核心操作:  翻牌找到匹配对。点击两张卡片，匹配成功则消除
成功判定:  在限时/限步内清除所有对
叙事绑定:  配对吸湿盒型号与区域、玻璃板尺寸与安装位、数据块与时间段
───────────────────────────────────────────────────────────
```

**难度爬升规则**:

| 参数 | Difficulty 1 | Difficulty 2 | Difficulty 3 |
|---|---|---|---|
| 配对数量 | 4 对 (8 卡) | 6 对 (12 卡) | 8 对 (16 卡) |
| 翻牌时间 | 无限制 | 无限制 | 无限制 |
| 步数限制 | 无限制 | 12 步 | 14 步 |
| 干扰相似度 | 低（完全不同的图标） | 中（同类但不同型号） | 高（极相似需仔细辨别） |
| 成功阈值 | 全部配对 | 全部配对 | 全部配对 |

**调优杠杆**:
- `pairCount`: 配对数量
- `stepLimit`: 步数限制（核心难度杠杆）
- `visualSimilarity`: 卡面视觉相似度
- `shuffleAlgorithm`: 洗牌策略（控制相邻布局）
- `revealDuration`: 翻牌展示时长

**边界情况**:
- 玩家快速连续点击同一张卡 → 忽略第二次点击
- 步数限制到达但未完成 → 自动判定失败，展示未配对卡片

**当前状态**: ✅ 实现。⚠️ 所有三个难度都没有步数限制（核心难度杠杆未启用）。diff3 在 Task 5 中使用，但对玩家的压力仅来自配对数量。

### 2.4 Timing（时机按键）

```
───────────────────────────────────────────────────────────
操作隐喻:  注浆确认 / 快门拍摄 / 功率锁定 / 设备重启
核心操作:  在移动指标进入目标区域时按下按钮
成功判定:  按键时刻与目标窗口的重合度 ≥ 阈值
叙事绑定:  在正确时机确认操作——早了冒失，晚了错过
───────────────────────────────────────────────────────────
```

**难度爬升规则**:

| 参数 | Difficulty 1 | Difficulty 2 | Difficulty 3 |
|---|---|---|---|
| 目标窗口大小 | 宽（占总行程 25%） | 中（15%） | 窄（8%） |
| 指标移动速度 | 慢（单程 3 秒） | 中（2 秒） | 快（1.2 秒） |
| 运动模式 | 匀速来回 | 变速（靠近窗口减速） | 随机变速 |
| 干扰因素 | 无 | 轻微画面抖动 | 强抖动 + 假窗口 |
| 成功阈值 | 1 次命中 | 1 次命中 | 2 次命中 (diff3 仅 T1S1C 使用) |

**调优杠杆**:
- `targetWindowSize`: 目标区域宽度
- `indicatorSpeed`: 指标移动速度
- `motionCurve`: 运动模式（线性/缓动/随机）
- `distractions`: 视觉干扰
- `hitCount`: 需要命中的次数

**边界情况**:
- 玩家在指标远离目标区时按键 → 判断为 miss，不扣血但浪费一次机会
- 高帧率 vs 低帧率的一致性 → 使用 deltaTime 而非帧计数

**当前状态**: ✅ 实现。⚠️ diff2→diff3 的目标窗口跳跃过大（15%→8%），可能让玩家感觉"不公"。

### 2.5 Calibrate（精密校准）

```
───────────────────────────────────────────────────────────
操作隐喻:  调校传感器 / 配比试剂 / 设置参数
核心操作:  将多个通道/滑块调整到目标值
成功判定:  所有通道的偏差在容差范围内
叙事绑定:  校准湿度传感器、调配化学试剂浓度、设置除湿机功率、调校芯片频率
───────────────────────────────────────────────────────────
```

**难度爬升规则**:

| 参数 | Difficulty 1 | Difficulty 2 | Difficulty 3 |
|---|---|---|---|
| 通道数 | 1 个 | 2 个 | 3 个 |
| 目标值精度 | 粗（±5%容差） | 中（±3%容差） | 细（±1.5%容差） |
| 滑块灵敏度 | 低（大步进） | 中 | 高（微小移动=大变化） |
| 时间限制 | 无 | 15 秒 | 10 秒 |
| 目标值可见性 | 始终可见 | 闪烁显示 | 记忆后隐藏 |

**调优杠杆**:
- `channelCount`: 同时校准的通道数
- `tolerance`: 允许的偏差范围
- `sensitivity`: 滑块灵敏度
- `timeLimit`: 时间压力
- `targetVisibility`: 目标值的可见模式

**边界情况**:
- 玩家在时间压力下大幅调整 → 可能出现 overshoot 振荡
- 通道间的互扰 → diff3 可引入：调整通道A会影响通道B的读数（仿真设备耦合）

**当前状态**: ✅ 实现。⚠️ 所有 diff3 的 calibrate 都使用"目标值隐藏"模式，但不同场景（湿度计 vs 芯片频率）的叙事合理性不同。芯片频率隐藏目标值不合理——玩家应该看到频率上限值。

---

### 2.6 Wire（接线连接）

```
───────────────────────────────────────────────────────────
操作隐喻:  连接线路 / 端口匹配 / 论点-数据关联
核心操作:  将左侧端口与右侧端口正确连接
成功判定:  所有连接正确
叙事绑定:  连接备用电源线路、关联建议论点与数据支撑
───────────────────────────────────────────────────────────
```

**难度爬升规则**:

| 参数 | Difficulty 1 | Difficulty 2 | Difficulty 3 |
|---|---|---|---|
| 端口对数 | 3 对 | 5 对 | 7 对 |
| 颜色编码 | 明显（红→红） | 弱化（同色系） | 无（全部同色） |
| 干扰端口 | 0 | 1 个假端口 | 2 个假端口 |
| 连线交叉 | 不交叉 | 轻微交叉 | 高度交叉 |
| 撤销次数 | 无限 | 2 次 | 0 次 |

**调优杠杆**:
- `portCount`: 端口对数量
- `colorCoding`: 颜色提示程度
- `decoyPorts`: 假端口数量
- `crossingLevel`: 连线交叉复杂度
- `undoLimit`: 可撤销次数

**边界情况**:
- 玩家连接了所有端口但有错误 → 显示"X/Y 正确"，不透露具体哪个错了（增加认知负荷）
- 移动端触屏 → 端口间距需 ≥ 44px（符合触摸目标标准）

**当前状态**: ✅ 实现。仅在 Task 4 Step 1 (diff2, 4个端口) 和 Task 5 Step 2 (diff3, 建议论点连线) 中使用。⚠️ 使用频率最低，玩家可能忘记这个操作类型。

### 2.7 Sequence（顺序排列）

```
───────────────────────────────────────────────────────────
操作隐喻:  时间线整理 / 流程编排 / 汇报结构
核心操作:  将散乱的条目拖拽排列为正确顺序
成功判定:  全部条目位置正确
叙事绑定:  排列巡检档案时间线、排序吸湿包放置优先级、组织汇报章节
───────────────────────────────────────────────────────────
```

**难度爬升规则**:

| 参数 | Difficulty 1 | Difficulty 2 | Difficulty 3 |
|---|---|---|---|
| 条目数量 | 4 个 | 6 个 | 8 个 |
| 正确顺序线索 | 明确（标题含时间） | 隐晦（需逻辑推断） | 需跨条目推理 |
| 拖拽反馈 | 即放即判定 | 全部确认后判定 | 全部确认后判定 |
| 时间限制 | 无 | 20 秒 | 15 秒 |
| 部分正确提示 | 显示正确数 | 不显示 | 不显示 |

**调优杠杆**:
- `itemCount`: 需要排序的条目数
- `clueExplicitness`: 正确顺序的线索明显程度
- `feedbackMode`: 即时反馈 vs 批量判定
- `timeLimit`: 时间限制
- `partialHint`: 是否显示"已经对了 X 个"

**边界情况**:
- 玩家拖拽条目到同位置 → 忽略操作
- 两个条目逻辑上都可能正确 → 确保只有一个"正确"顺序，避免歧义

**当前状态**: ✅ 实现。在 Task 2 (吸湿包顺序)、Task 4 Step 2 (AI 数据片段)、Task 5 Step 1/2 (档案/汇报) 中使用。⚠️ 是唯一一个用于"汇报/整理"场景的类型，差异化不足。

## 3. 难度爬升统一标准

### 3.1 难度定义

| 难度 | 目标玩家状态 | 期望通过率 | 典型叙事场景 |
|---|---|---|---|
| **Diff 1** | "我在学习" | 85-95% | 教学关、日常巡检操作 |
| **Diff 2** | "我在执行" | 65-80% | 标准任务、有压力的场景 |
| **Diff 3** | "我在极限操作" | 40-60% | 危急时刻、终章挑战 |

### 3.2 难度参数映射表

| 小游戏类型 | Diff 1→2 关键变化 | Diff 2→3 关键变化 |
|---|---|---|
| Trace | +2 段路径, -2s 时间, +10%阈值 | +2 段路径, -1s 时间, +10%阈值, +15px 振幅 |
| Memory | +2 项, -2s 展示 | +2 项, -1s 展示, +1 维度 |
| Match | +2 对, +步数限制 | +2 对, 收紧步限, +视觉相似度 |
| Timing | -10%窗口, -1s 单程 | -7%窗口, +变速, +1 命中 |
| Calibrate | +1 通道, -2%容差, +时间限制 | +1 通道, -1.5%容差, -5s, +目标隐藏 |
| Wire | +2 对, +1 假端口, -颜色提示 | +2 对, +1 假端口, 无色, -撤销 |
| Sequence | +2 条目, 隐晦线索, +时间 | +2 条目, 跨条目推理, -5s, 无提示 |

### 3.3 难度墙检测

当连续两步的小游戏难度跨越 > 1 级时，触发难度墙警告。

**当前项目中的难度墙**:

| 位置 | 前一步 | 当前步 | 跳跃幅度 | 建议 |
|---|---|---|---|---|
| Task 3: Step 1→Step 2 | diff1 memory | diff3 match | +2 🔴 | 将 diff3 改为 diff2，终章再用 diff3 |
| Task 5: Step 1→Step 2 | diff1 sequence | diff2 sequence / diff3 wire | +1 (OK) | OK，但 diff1 出现得不和谐 |

## 4. 叙事绑定指南

### 4.1 绑定公式

```
好的叙事绑定 = 操作隐喻 + 具体场景 + 风险暗示

  操作隐喻: "你手持红外热像仪"（解释为什么你在做描摹）
  具体场景: "正在精确标定壁画潮湿蔓延的边界线"（建立空间想象）
  风险暗示: "屏住呼吸，让扫描仪的红色激光沿着水渍边缘缓缓移动"（制造紧张）
```

### 4.2 叙事绑定质量评估

| 小游戏实例 | 操作隐喻 | 具体场景 | 风险暗示 | 综合 |
|---|---|---|---|---|
| T1S1A (trace, 红外扫描) | ✅ 手持热像仪 | ✅ 边界线 | ✅ 屏住呼吸 | ⭐⭐⭐ |
| T2S1C (trace, 手动采样) | ✅ 吸水纸擦拭 | ✅ 等高线 | ✅ 手电忽明忽暗 | ⭐⭐⭐ |
| T3S2C (match, 玻璃墙) | ✅ 配尺寸 | ✅ 各块玻璃 | ✅ 压坏壁画 | ⭐⭐⭐ |
| T5S2B (memory, 口头汇报) | ⚠️ 模糊 | ⚠️ "脑中搜索" | ❌ 无 | ⭐ |
| T4S2B (timing, 上线确认) | ✅ 确认按钮 | ⚠️ 模糊 | ❌ 无 | ⭐⭐ |

**改进 T5S2B**:
```
当前: "你在脑中搜索着那些数字。数据太多，能说对的只有最重要的那几个。"
改进: "主任盯着你，手指在桌面上有节奏地敲击。你报出的每一个数字都是信用的赌注——说对三个关键指标，你的汇报就立住了。说错一个，他会追问到底。"
```

## 5. 技能门槛与天花板

### 5.1 各小游戏的技能门槛

| 类型 | 核心技能 | 新手友好度 | 高手天花板 | 残疾可访问性 |
|---|---|---|---|---|
| Trace | 手眼协调 | ⭐⭐⭐⭐ | 极高（完美描摹需要外科医生级稳定） | ⚠️ 运动障碍玩家困难 |
| Memory | 短期记忆 | ⭐⭐⭐ | 中（记忆策略可训练） | ✅ 无时间限制可缓解 |
| Match | 模式识别 | ⭐⭐⭐⭐ | 低（运气成分大） | ✅ 纯认知操作 |
| Timing | 反应速度 | ⭐⭐⭐ | 高（可通过节奏感训练提升） | ⚠️ 反应慢的玩家可能持续失败 |
| Calibrate | 精细控制 | ⭐⭐ | 极高（微调可达亚像素精度） | ⚠️ 同 Trace |
| Wire | 逻辑推理 | ⭐⭐⭐⭐ | 中（排除法可解） | ✅ 纯认知操作 |
| Sequence | 逻辑排序 | ⭐⭐⭐ | 中 | ✅ 纯认知操作 |

### 5.2 Catch-Up 机制

**当玩家连续失败时**（当前未实现，建议添加）:

```
连续失败 2 次 → 第 3 次小游戏的 difficulty 临时 -1
              → HUD 显示 "设备自动校准中..."（叙事包装）
              → 但该次成功的奖励减半

连续失败 3 次 → 自动跳过小游戏（视为"操作被NPC接手"）
              → 选择效果减半（不获得 successDeltas，也不承受 failDeltas）
              → 最小化挫败感
```
