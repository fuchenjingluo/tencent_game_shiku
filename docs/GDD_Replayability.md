## 1. 当前状态评估

### 1.1 复玩性评分

| 维度 | 当前得分 | 目标得分 | 差距 |
|---|---|---|---|
| 分支可见性 | ⭐⭐ | ⭐⭐⭐⭐ | 玩家不知道有多少种结局/路径 |
| 收集驱动 | ⭐ | ⭐⭐⭐⭐ | 无成就、无收集品、无解锁 |
| 元进度 | ⭐ | ⭐⭐⭐ | 无 NG+、无跨周目奖励 |
| 技能精通 | ⭐⭐ | ⭐⭐⭐⭐ | 小游戏有 mastery 空间但无对应奖励 |
| 社交驱动 | ⭐ | ⭐⭐⭐ | 无排行榜、无分享机制 |
| 叙事好奇心 | ⭐⭐⭐ | ⭐⭐⭐⭐ | gameFlags 因果链提供了好奇心但没被"曝光" |

### 1.2 玩家流失点分析

```
                    进入游戏 (100%)
                         │
                    完成 Task 1 (95%)   ← 教学关流畅
                         │
                    完成 Task 2 (85%)   ← 预算压力开始
                         │
                    完成 Task 3 (70%)   ← 风险爆炸 + 预算枯竭
                         │
                    完成 Task 4 (55%)   ← 难度爬升
                         │
                    完成 Task 5 (50%)   ← 首次完成
                         │
                    看到评级报告 (50%)
                         │
                 再次挑战 ← ？？？ (5-10%)   🔴 最大流失点
```

> **核心问题**: 50% 的玩家完成了游戏，但只有 5-10% 点了"再次挑战"。  
> 原因：他们不知道"再来一次会有什么不同"。评级报告的差异化不够——玩家拿到了 B 或 C，但不知道如何变 A 或 S。

## 2. 多结局系统

### 2.1 结局判定逻辑

当前：只有评级（S/A/B/C/D）+ 基于 gameFlags 的评语生成。

**增强方案**: 引入 **6 个结局标签**，基于玩家的选择风格和关键 gameFlags：

| 结局 ID | 名称 | 触发条件 | 叙事基调 |
|---|---|---|---|
| `guardian` | **石窟守护者** | 全专业路线 + S/A 评级 | "你是真正的守护者。每一个操作都精准、冷静、科学。" |
| `pioneer` | **激进改革者** | 全激进路线 + 风险全程 ≤ 30 | "你用冒险换来了速度。石窟暂时安全了，但账本上有几个问题需要回答。" |
| `survivor` | **勉力维持者** | 全妥协路线 + 顽强完成 | "你尽了最大努力。石窟撑过了今天——但明天呢？" |
| `archivist` | **档案专家** | 证据 ≥ 70 + full_archive flag | "你的数据将成为未来十年所有修复决策的基石。" |
| `whistleblower` | **吹哨人** | 声誉 ≤ 25 + 完成所有任务 | "你做对了所有文物保护的事，但没人感谢你。有时候真理不需要掌声。" |
| `fallen` | **石窟之殇** | 风险 ≥ 85 + 任何路径完成 | "石窟挺过了千年风雨，却在你值守的这一天，留下了一道永久的伤痕。" |

**优先判定规则**: 如果同时满足多个结局条件，按 `fallen > guardian > archivist > pioneer > whistleblower > survivor` 优先顺序。

### 3.2 成就 UI 位置

在标题画面增加"🏆 成就"按钮，展示解锁进度（如 5/18）。

**关键**: 成就名称和描述本身就应该暗示不同的玩法策略，成为"非强制引导"。

## 4. 元进度与 New Game+

### 4.1 NG+ 继承机制

> 原则: NG+ 应该是"更丰富的体验"而非"更容易的体验"。

| 继承项 | 是否继承 | 理由 |
|---|---|---|
| 已解锁结局 | ✅ 继承 | 玩家要知道自己收集了多少结局 |
| 成就进度 | ✅ 继承 | 跨周目累计 |
| 属性值 | ❌ 不继承 | 保留每个周目的挑战性 |
| gameFlags | ❌ 不继承 | 每个周目是全新的因果链 |
| **可选初始加成** | ✅ 继承 | 见下方 |

### 4.2 NG+ 初始加成（三选一）

完成一个周目后，下一局开始时可在以下加成中选择一个：

| 加成 | 名称 | 效果 |
|---|---|---|
| 🛡️ | **前辈的经验** | 初始声誉 +10，初始风险 -10 |
| 📚 | **遗留的档案** | 初始证据 +10 |
| 💰 | **结转经费** | 初始预算 +3 |

这些加成让 NG+ 的早期体验更舒适，但不会根本上改变中后期的挑战。

### 4.3 周目标记

每次通关后，在标题画面显示：

```
┌──────────────────────────────┐
│    石窟守护者                 │
│                              │
│    第 3 次巡检                │
│    已解锁: 4/6 结局          │
│    成就: 11/18               │

│                              │
│    [🔄 重新挑战]              │
│    [📊 巡检员档案]            │
└──────────────────────────────┘
```

## 5. 挑战模式

### 5.1 速通模式

```
规则:
  - 计时器在游戏开始时启动
  - 跳过所有对话动画（自动快进）
  - 小游戏时间限制不变（给速通玩家保留操作挑战）
  - 结束时显示通关时间

排行榜:
  - 本地最快时间
  - 分级: 青铜 >20min / 白银 15-20min / 黄金 12-15min / 钻石 <12min
```

### 5.2 铁人模式

```
规则:
  - 无存档/读档（只有自动存档在退出时触发）
  - 小游戏失败 = 任务失败？→ 不，改为小游戏失败不可重来
  - 无属性转化冷却提示
  - 无 HUD 帮助提示

吸引力: 为完成"零失误"成就的硬核玩家准备
```

### 5.3 每日巡检

```
规则:
  - 每天生成一个随机种子
  - 随机化: 5 个任务中各步骤的 Choice 权重偏移
  - 例如: 今天 Task 1 Step 1 的 B 选项额外 +3 证据
  - 全服玩家同一天面对相同的种子
  - 排行榜: 当日评级前 10 名

价值：
  - 给老玩家每日回来的理由
  - 创造"刷新记录"的微型目标
  - 促进社区讨论策略
```

### 5.4 混沌模式（高复玩性）

```
规则:
  - 所有选择的效果数值 × 随机系数(0.8~1.2)
  - 转化系统可用次数翻倍但成本随机浮动
  - 风险爆炸触发率 +10%
  - 小游戏随机出现"稀有版"（完成获得双倍奖励）

吸引力: "永远不知道下一局会发生什么"
```

## 6. 实施路线图

### Phase 1: 最小可行复玩性（1-2 周）

| 优先级 | 项目 | 工作量 | 影响 |
|---|---|---|---|
| 🔴 P0 | 结局标签系统（6 个结局） | 中 | 给"再来一次"以理由 |
| 🔴 P0 | 评级报告增强（结局展示 + 风格分析） | 小 | 让通关体验更完整 |
| 🟡 P1 | NG+ 三选一加成 | 小 | 降低二周目前期压力 |
| 🟡 P1 | 周目计数器 + 结局解锁追踪 | 小 | 可视化进度 |

### Phase 2: 成就与探索（2-3 周）

| 优先级 | 项目 | 工作量 | 影响 |
|---|---|---|---|
| 🟡 P1 | 18 个成就的实现 + UI | 中 | 给玩家短期目标 |
| 🟢 P2 | 隐藏交互点（3 个） | 小 | 鼓励探索 |
| 🟢 P2 | gameFlags 曝光系统（在评级报告中显示"未走的路径"） | 中 | 揭示分支可能性 |

### Phase 3: 挑战与社区（3-4 周）

| 优先级 | 项目 | 工作量 | 影响 |
|---|---|---|---|
| 🟢 P2 | 速通模式 + 计时器 | 小 | 给硬核玩家目标 |
| 🔵 P3 | 每日巡检 + 种子系统 | 大 | 日活跃度引擎 |
| 🔵 P3 | 铁人模式 | 中 | 深度玩家留存 |
| 🔵 P3 | 混沌模式 | 中 | 内容裂变 |
